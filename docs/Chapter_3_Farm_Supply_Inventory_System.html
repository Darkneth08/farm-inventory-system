Chapter 3: User Interface Manual and Feature Highlights

Farm Supply Inventory System

X. User Interface Manual

The Farm Supply Inventory System does not use one generic dashboard for all users. It uses a shared application shell with role-based landing views and role-based navigation. This chapter follows the actual interface flow of the current system.

1. Opening the System

  1. Open the Farm Supply Inventory landing page in a browser.
  2. Review the public sections such as Features, Why It Works, FAQ, Get Started, and Contact.
  3. Click Log In to open the login form or Create Account to register.

Public components include the landing navigation bar, hero section with product preview, feature cards, and login or registration actions.

2. Logging In

  1. Enter email and password.
  2. Click Login.
  3. Wait for the system to validate the credentials.

Valid credentials open the protected app shell, while invalid credentials show an error alert. The first screen depends on role: customer users open Customer Home, staff users open Staff Desk, and manager, admin, and super_admin users open Dashboard.

3. Shared Application Shell

After authentication, the user enters the main app shell.

  • Left sidebar with brand title, subtitle, and role-mode pill
  • Role-aware menu buttons
  • Top header with current view title and subtitle
  • Current user information box
  • Refresh button
  • Logout button

The sidebar can show the following labels depending on role: Dashboard, Staff Desk, Home, Shop, My Orders, Favorites, Profile, POS or POS Counter, Operations, Reports or Shift Reports, Users, and Super Admin.

4. Admin Dashboard

The real Dashboard screen is used for manager, admin, and super admin workflows.

  • Admin overview guide panel
  • Shortcut buttons such as Open Stock Tasks and Open Reports
  • Workflow strip: Watch, Update, Review
  • KPI cards for Products, Active Products, Low Stock, Out Of Stock, Inventory Value, Today Sales, Month Sales, Total Suppliers, and Total Branches
  • Low-Stock Watch table
  • Recent Activity list
  • Top Movers, Stock by Category, and Monthly Sales tables

Users start by reviewing stock risk and recent movement, then open Operations for stock work or Reports for deeper review.

5. Staff Desk

The real default staff landing view is Staff Desk.

  • Staff Operations Hub hero panel
  • Open POS button
  • Shift Snapshot
  • Next steps
  • Metric cards for active products, low-stock alerts, today sales, and branches
  • Low-Stock Watch
  • Recent Movement
  • POS Snapshot

This page reduces clutter and moves staff quickly into checkout, stock checking, and receiving tasks.

6. Customer Home

The real default customer landing view is Customer Home.

  • Mini navigation: Overview, Steps, Quick Tools, Delivery
  • Welcome hero and action buttons
  • Metrics for Products, Cart, Active Orders, and Favorites
  • Delivery profile card
  • Start Here panel
  • Main customer tools panel
  • Delivery Details panel

This home screen prepares the customer before opening the full customer workspace.

7. Customer Workspace

The Customer Workspace uses four real sections: Browse, Orders, Favorites, and Profile.

7.1 Browse

The Browse section is divided into the real workflow cards 1. Browse products, 2. Review product, and 3. Checkout.

  • Catalog search input
  • Stock-status filter
  • Sort dropdown
  • Product cards with View, Favorite, and Add to Cart
  • Product spotlight card with price, category, supplier, stock, and customer reviews
  • Cart panel with quantity controls
  • Payment method selector: COD, Cash, Online Payment
  • Optional promotion code and delivery notes
  • Place COD order, Place cash order, or Place online order

7.2 Orders

The Orders section contains tracking summary chips, live tracking cards, an order table, a request form, and a request history table. Customers can review order status, reorder previous items, and submit product requests.

7.3 Favorites

The Favorites section contains Saved for later, Low-stock alerts, Updates, and Seasonal picks. It keeps saved products, alerts, notifications, and suggestions together.

7.4 Profile

The Profile section contains the profile form, password form, and product review form so delivery details remain ready for checkout.

8. POS Counter

The real POS screen is the dedicated staff sales transaction view.

  • POS Workspace hero
  • Live warehouse label and stock-source note
  • Step strip: Select warehouse, Add products, Complete payment
  • Warehouse selector and product search
  • Live product catalog cards
  • Checkout form
  • Discount and payment controls
  • Sale cart with editable quantity inputs
  • Summary chips for items, subtotal, discount, grand total, amount tendered, balance due, change, and payment status
  • Complete Sale button
  • Sale result card
  • Print Last Receipt
  • Recent POS Sales

This is the actual main transaction screen used for counter checkout.

9. Operations Workspace

The Operations view is organized into the real tabs Setup, Product Catalog, Stock Adjustment, Inventory Summary, Batch Health, Receiving, Orders, and Branches.

  • Setup: category, supplier, and warehouse forms
  • Product Catalog: product form, catalog list table, and quick stock in/out form
  • Stock Adjustment: presets, draft summary chips, adjustment form, draft table, and posted history
  • Inventory Summary: inventory KPI chips, status breakdown, and warehouse summary table
  • Batch Health: filters, status update form, aging metrics, batch table, and aging table
  • Receiving: supply deliveries form, draft delivery items, and recent stock receipts
  • Orders: order management and promotions
  • Branches: smart forecasting, multi-branch management, and branch inventory overview

10. Reports Workspace

The Reports view is organized into the real report groups Inventory, Sales, Access Logs, and Settings.

  • Inventory: Movement Summary and Expiring Stock
  • Sales: Sales Report, Top Selling Products, Inventory Value, and Low Stock Report
  • Access Logs: Login Activities and Audit Logs
  • Settings: Settings & Backup and Safeguards

11. Users Workspace

The Users screen uses two real tabs: Team Directory and Editor.

  • Account filter form
  • User table with Edit and Delete
  • Account creation and editing form
  • Role selection for customer, staff, manager, admin, and super_admin

12. Super Admin Workspace

The Super Admin view is organized into the real control areas Overview, Setup, Product Catalog, Stock Flow, Inventory Summary, User Accounts, Security, and Logs.

Typical components include KPI overview cards, role distribution tables, shortcut cards, setup counts, product and stock summary cards, user-account actions, security tools, and log or activity panels.

13. Logging Out

  1. Click Logout from the top header.
  2. Wait for the session to close.
  3. The system returns the user to the public landing view.
